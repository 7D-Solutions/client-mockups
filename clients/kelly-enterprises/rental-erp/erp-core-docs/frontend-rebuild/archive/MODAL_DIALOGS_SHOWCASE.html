Modal Features Summary

Common Features Across All Modals

  • Portal Rendering: All modals render outside the normal DOM hierarchy
  • Keyboard Support: Escape key to close, Tab navigation
  • Backdrop Click: Click outside to close (can be disabled)
  • Loading States: Button loading indicators during async operations
  • Form Validation: Required field indicators and error messaging
  • Accessibility: ARIA attributes, focus management, screen reader support
  • Toast Integration: Success/error notifications after operations
  • Responsive Design: Mobile-friendly layouts

Modal Management Patterns

1. Modal Managers

Centralized state management for related modals:

  • ModalManager - Infrastructure modal coordination
  • GaugeModalManager - Gauge-related modal orchestration

2. Nested Modals

Complex workflows with modal-in-modal patterns:

  • QC Approvals ‚Üí Reject Modal
  • Edit Gauge ‚Üí Confirmation Dialog
  • Unseal Requests ‚Üí Unseal Confirm

3. Event-Driven Modals

Using event bus to prevent circular dependencies:

  • Password change modal triggered by events
  • Cross-module modal communication
