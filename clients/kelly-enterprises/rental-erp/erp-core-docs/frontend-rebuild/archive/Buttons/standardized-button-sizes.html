Standardized Button Sizes

Consistent sizing system for modern button implementation

Current Button Size Issues

⚠️ Issue: Inconsistent Font Sizes

Small and Medium buttons both use 14px font size, making them hard to distinguish

⚠️ Issue: Limited Size Options

Only 3 sizes available (sm, md, lg) - missing extra small for compact UI and extra large for hero CTAs

⚠️ Issue: Inconsistent Scaling

Padding doesn't scale proportionally with font size

Proposed Standardized Sizes

✅ Solution: 5-Size System with Consistent Scaling

Each size has distinct visual hierarchy with proportional padding and font sizes

Extra Small (xs)
12px font • 6px/12px padding • 28px height
Small (sm)
14px font • 8px/16px padding • 36px height
Medium (md)
16px font • 10px/20px padding • 40px height
Large (lg)
18px font • 12px/24px padding • 48px height
Extra Large (xl)
20px font • 16px/32px padding • 56px height

Use Case Guidelines

Extra Small (xs)

Table row actions, compact toolbars, inline form buttons, pagination

Small (sm)

Secondary actions, modal footers, form groups, navigation

Medium (md) - Default

Primary actions, form submissions, standard CTAs

Large (lg)

Important actions, page headers, empty states

Extra Large (xl)

Hero sections, landing pages, onboarding flows

Implementation Guide

TypeScript Interface Update

interface ButtonProps {
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl';  // Extended from 3 to 5 sizes
  // ... other props
}

Usage Examples

// Extra small - for tight spaces
<Button size="xs" variant="secondary">Edit</Button>

// Small - for secondary actions
<Button size="sm" variant="danger">Delete</Button>

// Medium (default) - for primary actions
<Button size="md" variant="primary">Save Changes</Button>

// Large - for important CTAs
<Button size="lg" variant="success">Complete Order</Button>

// Extra large - for hero sections
<Button size="xl" variant="primary">Start Free Trial</Button>

Migration Strategy

Size Mapping

  • ✓ Current sm → New xs or sm (context-dependent)
  • ✓ Current md → New md (no change)
  • ✓ Current lg → New lg (slightly adjusted)

Benefits

  • ✓ Clear visual hierarchy with 5 distinct sizes
  • ✓ Consistent scaling ratio between sizes
  • ✓ Better accessibility with minimum touch targets
  • ✓ Flexibility for different UI contexts
  • ✓ Modern design system alignment
