Frontend UI Comparison - Phase 1: Buttons

✅ Comprehensive Search Performed: This analysis includes ALL button implementations found through exhaustive search of both Legacy (.css, .tsx) and Modular (.module.css, .tsx) systems, including className patterns, button elements, and onClick handlers.

Analysis by Instance 1 - Complete comparison of button implementations between Legacy and Modular systems

Executive Summary

Critical Finding: The modular system has THREE different button implementation approaches running simultaneously:
  • 1. The shared Button component (infrastructure/components/Button.tsx)
  • 2. Direct button elements with CSS Module classes (e.g., GaugeRow.module.css)
  • 3. Legacy-style class names in some components
This indicates an incomplete migration with inconsistent patterns.
Important: Legacy system has 20+ unique button styles, while the modular Button component only covers 8 variants. Many specialized buttons (login, logout, close, status buttons) are not available in the shared component.
Discovery Stats:
  • Legacy: 46 files with button usage, ~20 unique button classes
  • Modular: 102 files with button references, but only partial use of shared component
  • Migration completion: ~30% (most components still use custom implementations)

Gap Analysis: Missing in Modular

Missing Button Variants in Button Component:

  • login-btn: Full-width login button with specific styling
  • logout-btn: Red logout button (using generic danger instead)
  • close-btn: Icon-only close button for modals
  • icon-btn: Small utility buttons with minimal styling
  • status-btn: Gradient/special styling for status indicators
  • Specialized gauge buttons: checkout, checkin, transfer variants

Inconsistent Implementation:

  • GaugeRow.tsx uses CSS Module classes instead of Button component
  • Some modals use Button component, others use custom classes
  • No standard pattern for icon-only buttons (close, edit icons)
  • Loading states implemented differently across components

New Features in Modular (not in Legacy):

  • Built-in loading state with spinner
  • Icon prop with automatic spacing
  • TypeScript support and props validation
  • Focus ring styling
  • Outline variant
  • Better prop forwarding (...props)
