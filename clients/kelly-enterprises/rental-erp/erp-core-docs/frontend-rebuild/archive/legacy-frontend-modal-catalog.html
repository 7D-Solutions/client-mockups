Modal System Summary

Architecture Overview

Modal Categories

  • User Management (4): Add User, User Details, Password, Reset Password
  • Gauge CRUD (4): Create Gauge, Edit Gauge, Edit Gauge Form, Gauge Details
  • Workflow Modals (10): Checkin, Checkout (via Confirm), Transfer (3 variants), Transfer Cancel Confirm, Review, Sealed Notice, Unseal Confirm
  • Utility Modals (4): Confirm, Reject, Bulk Update, QC Approval Dialog
  • Inline Dialogs (2): QC Approval (in Details), System Recovery Confirmation
  • Special Components (1): Transfer Receive Button (self-contained modal)

Common Patterns

  • Overlay click closes modal (with stopPropagation on content)
  • Form validation before submission
  • Loading states during async operations
  • Notification callbacks for user feedback
  • Role-based visibility and actions
  • Consistent button styling and placement

Modal Management Systems

Legacy System: The AllModals.tsx component serves as a central orchestrator for all modals in the application.

New System: The GaugeModalManager.tsx in the gauge-tracking module provides a more structured approach with:

  • Enum-based modal types for type safety
  • Context-based state management
  • Modal history for cascading flows
  • Centralized modal rendering

Modal Implementation Patterns

  • Standalone Components: Most modals are separate components in /modals folder
  • Inline Modals: QC Approval Dialog embedded in parent component
  • Self-Contained: TransferReceiveButton includes its own modal states
  • Generic Modals: Confirm Modal handles multiple action types
  • Alternative Versions: Some modals have both legacy and new implementations
