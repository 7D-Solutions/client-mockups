Location Display Options

Compare different approaches for displaying location hierarchy in inventory tables

Option 1: Single Hierarchy Column

Compact

Pros

  • Cleanest appearance
  • Minimal horizontal space
  • Easy to scan
  • Clear visual relationship

Cons

  • Can't sort/filter by building
  • Can't sort/filter by zone
  • Harder to find specific building

SET-0002 NPT 1/2-14 RING GO Gauge A5 Building 1 → Bulk Storage 1

Option 2: Separate Building & Zone Columns

Sortable

Pros

  • Can sort by building
  • Can sort by zone
  • Can filter each separately
  • Best for data analysis

Cons

  • More horizontal space
  • More visual clutter
  • Relationship less obvious

Option 3: Location + Building Only

Simplified

Pros

  • Good balance of detail
  • Can sort/filter by building
  • Less cluttered than Option 2
  • Building is most important

Cons

  • Loses zone information
  • Zone might be important
  • Not showing full hierarchy

Option 4: Location + Building/Zone (Toggle to Separate)

Flexible

Pros

  • User controls detail level
  • Compact by default
  • Can expand for sorting
  • Best of both worlds

Cons

  • More complex to implement
  • User has to learn feature
  • Table width changes
