Location Hierarchy Column Ordering

Two approaches for organizing location hierarchy columns

Option A: Broad-to-Specific (Context First)

RECOMMENDED
BUILDING
→
ZONE
→
LOCATION

Pros

  • Natural hierarchy flow (large → small)
  • Matches physical warehouse layout
  • Context before detail pattern
  • Better for grouping/organization
  • Industry standard approach

Cons

  • Location code not first (less prominent)
  • Eye travels further to reach location
Best for: Users who think "I need something in Building 1, Zone QC, Location L1" - hierarchical mental model

Option B: Specific-to-Broad (Detail First)

ALTERNATIVE
LOCATION
→
ZONE
→
BUILDING

Pros

  • Location code is first (most prominent)
  • Users see specific location immediately
  • Good for quick location lookup
  • Detail-first scanning pattern

Cons

  • Reversed hierarchy flow (small → large)
  • Context comes after detail
  • Less intuitive for grouping
  • Unusual for warehouse systems
  • Harder to mentally organize
Best for: Users who think "I need location L1, which happens to be in Building 1, Zone QC" - location-first mental model

Quick Comparison

Criteria Option A: Building → Zone → Location Option B: Location → Zone → Building
Hierarchy Flow ✓ Natural (broad → specific) ✗ Reversed (specific → broad)
Physical Layout Match ✓ Matches how warehouse is organized ✗ Doesn't match physical flow
Grouping/Sorting ✓ Easy to group by building, then zone ⚠ Harder to see building groups
Location Prominence ⚠ Location is last column ✓ Location is first (most visible)
Industry Standard ✓ Standard approach ✗ Unusual ordering
Mental Model ✓ "Navigate from large to small" ⚠ "Find specific, then see context"

Recommendation: Option A (Building → Zone → Location)

Option A follows the natural hierarchical flow from broad to specific, matching how physical warehouses are organized and how users mentally navigate locations. This is the industry standard approach used in professional warehouse management systems.
