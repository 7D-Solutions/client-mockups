üìê Component Architecture - Single Source of Truth
PermissionSelector.tsx (NEW - Shared Component)
Reusable permission selection UI component with role template support
  • Loads permissions from adminService.getPermissions()
  • Displays grouped permission checkboxes (Gauge, User, Certificate)
  • Optional role template quick-select
  • Collapsible permission groups
  • Permission count summary
Props Interface:
interface PermissionSelectorProps { selectedPermissions: string[]; onPermissionsChange: (permissions: string[]) => void; availableRoles?: Role[]; showRoleTemplates?: boolean; disabled?: boolean; }
‚¨áÔ∏è Used By ‚¨áÔ∏è
AddUserModal.tsx (MODIFIED)
Two-step wizard for creating users with permissions
  • Step 1: User details (no role selector)
  • Step 2: Embeds <PermissionSelector />
  • Validates at least 1 permission selected
  • Creates user + permissions atomically
Usage in Step 2:
<PermissionSelector selectedPermissions={selectedPermissions} onPermissionsChange={setSelectedPermissions} availableRoles={availableRoles} />
PermissionManagementModal.tsx (MODIFIED)
Edit permissions for existing users
  • Replaces existing permission UI
  • Uses same <PermissionSelector /> component
  • Loads user's current permissions
  • Updates permissions via PUT API
Usage in Modal:
<PermissionSelector selectedPermissions={userPermissions} onPermissionsChange={setUserPermissions} availableRoles={availableRoles} />
